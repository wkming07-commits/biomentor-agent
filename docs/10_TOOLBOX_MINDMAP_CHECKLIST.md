### 1.1 蛋白结构查看器

目标：从“示例蛋白展示”升级成“可自己搜索的蛋白结构学习工具”。

#### 功能清单

- [ ] 支持用户搜索蛋白名、基因名、PDB ID、UniProt ID。
- [ ] 输入 PDB ID（如 `4HHB`）时直接加载实验结构。
- [ ] 输入 UniProt ID（如 `P42212`）时尝试加载 AlphaFold 预测结构。
- [ ] 输入关键词（如 `GFP`、`TP53`、`insulin`、`Cas9`）时返回候选结果。
- [ ] 搜索结果以候选卡片显示：蛋白名、物种、结构来源、适合讲解的主题。
- [ ] 用户点击候选后加载 3D 结构。
- [ ] 3Dmol.js 结构区域具备加载中、成功、失败三种状态。
- [ ] 支持结构显示样式切换：cartoon / stick / sphere。
- [ ] 显示结构来源，但使用用户语言表达：实验结构 / 预测结构。
- [ ] AI 解释区随当前蛋白变化。

#### 教学清单

- [ ] 解释结构域。
- [ ] 解释活性位点或功能区域。
- [ ] 解释实验结构与预测结构的区别。
- [ ] 自动生成至少 1 个理解题或引导问题。

#### 验收例子

- [ ] 搜 `GFP` 能看到 GFP 候选和结构。
- [ ] 搜 `4HHB` 能看到血红蛋白结构。
- [ ] 搜不存在内容时显示友好空状态。
- [ ] 外部结构加载失败时页面不崩溃。

### 1.2 质粒图谱查看器

目标：从“静态质粒图”升级成“教学型质粒图谱分析器”。

#### 功能清单

- [ ] 内置经典质粒：pBR322、pUC19、pET-28a、pGEX。
- [ ] 每个内置质粒包含核心元件：ori、抗性基因、promoter、MCS、tag、terminator。
- [ ] 支持上传 GenBank `.gb` / `.gbk`。
- [ ] 支持上传 FASTA `.fa` / `.fasta`。
- [ ] GenBank 自动解析 `FEATURES`。
- [ ] FASTA 显示为未注释序列轨道，并给出自然语言提示。
- [ ] 环形图谱根据 feature 坐标绘制。
- [ ] 右侧显示元件列表。
- [ ] 点击元件时高亮图谱对应区域。
- [ ] 当前选中元件显示解释卡片。

#### 教学清单

- [ ] 解释 ori 决定复制宿主范围。
- [ ] 解释抗性基因用于筛选。
- [ ] 解释 promoter 控制表达。
- [ ] 解释 MCS 用于插入外源片段。
- [ ] 给出实验设计引导问题。

#### 验收例子

- [ ] 选择 pET-28a 能看到 T7 promoter、His-tag、KanR。
- [ ] 上传 GenBank 后能生成图谱。
- [ ] 点击 AmpR 后右侧解释抗性筛选。
- [ ] 上传 FASTA 后提示“未注释序列，可上传 GenBank 获取完整元件”。

### 1.3 序列分析工具

目标：从“数字分析”升级成“序列理解和实验检查工具”。

#### 功能清单

- [ ] 自动识别序列类型：DNA、RNA、Protein、Invalid / Mixed。
- [ ] 支持 FASTA header 清洗。
- [ ] 显示长度、GC、AT、碱基组成。
- [ ] 支持 DNA → RNA 转录。
- [ ] 支持生成反向互补链。
- [ ] 支持 DNA 翻译蛋白。
- [ ] 支持 ORF 查找。
- [ ] 检测常见酶切位点：EcoRI、BamHI、HindIII、XhoI、NdeI。
- [ ] 酶切位点在序列中高亮。
- [ ] 引物设计结果更完整：forward primer、reverse primer、Tm、GC、产物长度、风险提示。
- [ ] 分析结果随输入实时更新。

#### 教学清单

- [ ] 解释 GC 是否适合 PCR。
- [ ] 解释 ORF 是否完整。
- [ ] 解释酶切位点能否用于克隆。
- [ ] 解释引物是否合理。

#### 验收例子

- [ ] 输入 DNA 能识别为 DNA。
- [ ] 点击转录能得到 RNA。
- [ ] 点击反向互补能得到 reverse complement。
- [ ] 能找出最长 ORF。
- [ ] EcoRI / BamHI 位点能显示位置并高亮。

## 任务 2：思维导图模块

### 模块定位

思维导图模块不是普通树状脑图，也不是通路知识图谱。它定位为“学习路径组织器”：帮助学生知道自己在学什么、知识点之间是什么层级关系、下一步应该学什么、哪些工具可以辅助理解。

- 通路知识图谱：讲生物机制，强调激活、抑制、上下游关系。
- 思维导图模块：讲学习结构，强调主题、层级、掌握状态、推荐路径。

---

### 核心设计方向

采用“渐进展开式 BioMind Map”：

- [ ] 默认只显示中心节点和一级主题，不一次性铺开所有内容。
- [ ] 点击一级节点后展开其子节点。
- [ ] 点击二级节点后显示详细解释和推荐工具。
- [ ] 当前路径高亮，非当前分支降低透明度。
- [ ] 图上只放短词，详细内容放右侧面板。
- [ ] 节点状态通过颜色表达：已掌握、需复习、薄弱、推荐下一步、未学习。

### 节点详情面板内容

每个节点至少包含：

- [ ] 一句话解释。
- [ ] 需要掌握的关键点。
- [ ] 推荐工具入口。
- [ ] 推荐练习。
- [ ] 下一步学习建议。

示例：

```text
节点：质粒载体

一句话解释：
质粒是常用于基因克隆和表达的环状 DNA 载体。

你需要掌握：
1. ori 决定复制能力
2. 抗性基因用于筛选
3. MCS 用于插入外源 DNA

推荐工具：质粒图谱查看器
推荐练习：判断 pET-28a 中 T7 promoter、His-tag、KanR 的作用。
下一步：启动子 → 表达载体 → 克隆设计
```

- [ ] 使用 radial / orbit 风格，而不是传统树状图。
- [ ] 中心节点有柔和 glow。
- [ ] 分支线条从父节点“生长”出来。
- [ ] 点击展开有平滑过渡。
- [ ] 当前路径使用亮色高亮。
- [ ] 非当前分支淡出，不抢注意力。

- [ ] 背景可加入非常轻的分子网络 / DNA 曲线纹理，但不能喧宾夺主。

### 技术方案建议

第一版建议使用自定义 SVG + 极坐标布局，不引入重型编辑器：

- [ ] 数据放在 `frontend/lib/mindmap-data.ts`。
- [ ] 页面放在 `frontend/app/mindmap/page.tsx`。
- [ ] 使用 React state 管理 `selectedNodeId`、`expandedNodeIds`、`focusedPath`。
- [ ] SVG 画 edges，节点使用可点击 HTML/SVG 组合。
- [ ] 不做拖拽编辑，先做高质量浏览器。
